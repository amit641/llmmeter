**One line of code. Every LLM call instrumented.**

```ts
import { meter } from "@amit641/llmmeter";
const openai = meter(new OpenAI());
```

That's it. From now on every call is recorded with token counts, USD cost,
latency, time-to-first-token, prompt hash, and any custom context you attach
(`feature`, `userId`, `traceId`). Data goes to a local SQLite file and a
beautiful dashboard at `npx @amit641/llmmeter-cli dashboard`.

Production? Ship to your existing Postgres, OpenTelemetry stack, or our
self-hosted Docker collector. Hosted cloud is coming soon.

## Why llmmeter?

Most teams ship an LLM-powered feature and then discover, at the end of the
month, that it costs three times what they thought it would. There's no
breakdown by feature, no per-user attribution, no warning when a single
runaway agent burns through the budget. The vendor dashboards don't help —
you can see "$1,847 spent on `gpt-4o`" but you have no idea which user,
endpoint, or prompt is responsible.

`llmmeter` fixes that.
